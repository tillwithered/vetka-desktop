# Vetka Desktop: официальный Store-only сборщик и обновления

## Цель

Сделать Amazon-контур операционным: Vetka использует только официальные страницы Monster High Store и показывает по одной актуальной цене на регион для каждой куклы. Убрать из интерфейса дубли и технические логи браузера. Управление обновлениями перенести в настройки приложения.

## Scope

### Store-only сборщик

- Единственный источник каталога и текущих цен — официальные Monster High Store US, UK, DE, ES и IT.
- Ручная кнопка обновления запускает импорт Store по всем пяти регионам. Обычный запуск приложения не начинает Amazon-проверку.
- После ручного запуска приложение повторяет только Store-импорт раз в два часа, пока открыто.
- Не выполнять общий поиск Amazon, не сканировать произвольные ссылки и не добавлять сторонние предложения в текущие цены.
- Кнопки «Обновить цены» в карточке и избранном запускают тот же общий Store-импорт, а не индивидуальный поиск Amazon.

### Одна цена на регион

- Store-импорт сохраняет подтверждённый листинг для найденного ASIN и региона.
- Когда Store подтверждает цену региона, другие текущие листинги той же куклы в этом регионе перестают участвовать в `current`/таблице/карточке. История и сами записи не удаляются.
- Это устраняет двойные цены, включая два значения IT, не стирая данные заказов и историю цен.

### Ошибки и состояние сканирования

- Рабочий статус показывает фазу Store и регион/счётчик позиции.
- Сообщение об ошибке ограничено безопасным коротким текстом региона: блокировка, CAPTCHA, временная недоступность или сбой импорта.
- Командные строки Chromium, пути профиля и stack/log output никогда не отправляются в renderer и не выводятся в UI.

### Обновления

- Удалить всплывающий `UpdateNotification`-баннер из дерева renderer.
- Сохранить маленькую кнопку «Обновить» у версии в нижней части sidebar, когда обновление скачано.
- В настройках добавить ограниченную по ширине секцию «Приложение» с:
  - фактической версией из health IPC;
  - кнопкой «Проверить обновления»;
  - pending-состоянием;
  - понятным статусом «новая версия ищется / скачивается / готова»;
  - действием перезапуска только для уже скачанного пакета.
- Проверка обновлений не перезапускает приложение и не выводит баннер.

## Технический дизайн

1. `CatalogScanService` становится orchestration-сервисом Store-импорта: запуск вручную и повтор раз в два часа. Универсальный `PriceService.refreshCatalogEntry` не вызывается из каталожного цикла.
2. `OfficialStoreImportService` сохраняет результат каждой проверенной Store-карточки через новый атомарный метод репозитория: подтвердить Store-ASIN и вывести из текущего набора другие листинги той же пары `dollId + region`.
3. Worker преобразует ошибки Playwright в ограниченный набор безопасных сообщений до передачи в main. `CatalogScanService` сохраняет только это сообщение в `lastError`.
4. `PriceRepository.current` и `currentForDolls` дополнительно выбирают один последний подтверждённый листинг на `(doll, region)`, что защищает интерфейс от уже накопленных дублей.
5. Settings использует существующие IPC `health`, `updates.check`, `updates.getState` и `updates.restartAndInstall`; новых IPC или схем БД для UI обновлений не требуется.

## UI decision

- Поверхность: desktop settings page и существующий Maia/Violet shadcn `FormSection`.
- Задача пользователя: проверить наличие обновления, не перезапуская приложение, и увидеть понятный результат.
- Основное действие: компактная `Button size="sm"` «Проверить обновления» в секции «Приложение».
- Паттерны: существующие `FormSection`, `PageHeader`, `Button`, `Badge`, `Alert`; новая визуальная библиотека не добавляется.
- Desktop: секция ограничена существующей формой; нет полноширинных CTA, баннера или нового sidebar-элемента.
- Состояния: initial, checking, available/downloading, downloaded, error; для таблицы — long title и одна цена на регион.
- Отклонения: нет. Используются только токены, Lucide и существующие shadcn-примитивы.

## Проверки

- Репозиторий: Store-цена вытесняет текущий не-Store листинг только в том же регионе; история остаётся; `current` и `currentForDolls` не возвращают дубль региона.
- Scan service: ручной Store-only запуск, отсутствие общего scan, расписание два часа после ручного запуска, краткая безопасная ошибка.
- Worker: ошибка закрытого браузера не содержит executable path или Chromium arguments.
- Renderer: баннера нет; Settings отображает версию, проверку и состояния; таблица не показывает два значения IT.
- Финально: test, lint, typecheck, Windows package, инспекция Chromium resources и Electron QA на 1080×720, 1280×800, 1440×900.
